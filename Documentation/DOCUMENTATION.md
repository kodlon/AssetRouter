## Як це працює — просто

Уяви що у тебе є охоронець на вході до проєкту. Кожен раз коли ти кидаєш файл у Unity — охоронець дивиться на його назву і вирішує:

- **Знайоме ім'я** → кидає файл у правильну папку і застосовує готовий preset налаштувань
- **Незнайоме ім'я** → зупиняє тебе і питає що робити

Ти навчаєш охоронця через список правил. Правило — це по суті: *"якщо файл починається на `T_` — кинь його ось сюди і застосуй цей preset"*.

Все. Більше нічого вручну робити не треба.

## Три частини системи

Система складається з трьох незалежних частин, кожна зі своєю роллю:

```
┌─────────────────────────────────────────────────────────┐
│  ScriptableObject (.asset файл)                         │
│  — зберігає всі правила на диску                        │
│  — живе в проєкті, потрапляє в git разом з командою     │
└─────────────────────────────────────────────────────────┘
           ↑ читає та зберігає
┌─────────────────────────────────────────────────────────┐
│  Editor Window  (Tools > Asset Router Settings)         │
│  — зручний UI для керування правилами                   │
│  — додати / видалити / змінити правило без коду         │
└─────────────────────────────────────────────────────────┘
           ↓ читає правила
┌─────────────────────────────────────────────────────────┐
│  AssetPostprocessor  (працює у фоні, завжди активний)   │
│  — перехоплює кожен імпорт автоматично                  │
│  — застосовує preset і переміщує файл                   │
└─────────────────────────────────────────────────────────┘
```

**SO** — це файл збереження. Без нього правила зникали б при кожному перезапуску Unity.

**Window** — це інтерфейс. Ти спілкуєшся з системою через нього, не через код.

**Postprocessor** — це виконавець. Він постійно чергує у фоні і не вимагає жодних дій з твого боку.

## Що відбувається коли ти кидаєш файл

Розберемо на прикладі: ти кидаєш `T_Rock_D.png` у папку `Assets/`.

```
1. Unity починає імпорт файлу
        ↓
2. Postprocessor прокидається і дивиться в SO:
   "Є правило де prefix = T_?"
        ↓ Так, є — "General Textures"
3. Застосовує TextureImporter.preset через preset.ApplyTo(importer):
   compression, mip maps, max size — все що записано в preset
        ↓
4. Unity завершує імпорт
        ↓
5. Postprocessor перевіряє: файл у правильній папці?
   Зараз: Assets/T_Rock_D.png
   Треба:  Assets/Art/Textures/
        ↓ Не збігається
6. Переміщує файл → Assets/Art/Textures/T_Rock_D.png
        ↓
7. Unity переімпортує файл на новому місці
   (preset вже застосований — все ок)
```

Якщо б ти кинув `qwerty_texture.png` — на кроці 2 правило не знайшлось би, і замість кроків 3–7 з'явився б попап з питанням що робити.

## Як користуватись

### Відкрити

```
Tools > Asset Router Settings
```

### Вікно складається з трьох зон

**Верхня панель** — вибір активного SO (файлу з правилами). Зазвичай там вже є `ImporterSettingsDatabase`.

**Список правил** — всі поточні правила. Зліва у кожного:
- галочка — вмикає/вимикає правило без видалення
- коротке резюме: `назва [prefix*] → папка`

**Панель деталей** — з'являється знизу коли вибираєш правило зі списку. Тут редагуєш всі поля.

### Зберегти зміни

Кнопка **Save / Apply** внизу вікна. Не забувай натискати після кожної зміни.

## Як додати нове правило

1. Відкрий `Tools > Asset Router Settings`
2. Натисни `+` у правому нижньому куті списку правил — нове правило з'явиться в списку
3. Клікни на нього — знизу відкриються поля для редагування
4. Заповни `Prefix` і `Target Folder`
5. Прив'яжи `Import Preset` — перетягни `.preset` файл або обери через picker
6. Натисни **Save / Apply**

Готово. Тепер файли з цим префіксом оброблятимуться автоматично.

---

## Поля правила

| Поле | Що робить | Приклад |
|------|-----------|---------|
| `Rule Name` | Назва у списку, ні на що не впливає | `General Textures` |
| `Is Enabled` | Вимкнути правило тимчасово не видаляючи | ✓ / ✗ |
| `Prefix` | Ім'я файлу **починається** з... | `T_` |
| `Suffix` | Ім'я файлу (без розширення) **закінчується** на... | `_LOD0` |
| `Extension Filter` | Тільки файли з цим розширенням. Порожньо = будь-яке | `.png` |
| `Target Folder` | Куди перемістити файл після імпорту | `Assets/Art/Textures/` |
| `Import Preset` | Unity Preset файл з налаштуваннями імпортера | `TextureImporter.preset` |

> Файл спрацьовує по правилу тільки якщо **всі** задані умови виконані одночасно. Якщо поле порожнє — ця умова ігнорується.

> Якщо цільова папка не існує — вона створюється автоматично.

## Presets

**Що таке Import Preset** — Unity asset (`.preset` файл) що зберігає знімок всіх налаштувань імпортера. Створюється через Inspector будь-якого ассету: виділи файл → у правому верхньому куті Inspector натисни іконку Presets → "Save current to...".

**Presets в проекті** (`Assets/AssetRouter/Presets/`):

| Файл | Для чого |
|------|----------|
| `TextureImporter.preset` | Загальні текстури: 2048px, compressed, mipmaps ON |
| `TextureImporter_UI.preset` | UI: 1024px, compressed, mipmaps OFF |
| `AudioImporter.preset` | SFX: Compressed In Memory, Vorbis |
| `AudioImporter_Music.preset` | Музика: Streaming, Load In Background |

**Як додати новий preset:**
1. Імпортуй будь-який ассет потрібного типу
2. Налаштуй його в Inspector як треба
3. Натисни іконку Presets → "Save current to..." → збережи в `Assets/AssetRouter/Presets/`
4. Прив'яжи до правила в Asset Router Settings

**Важливо:** preset прив'язаний до типу імпортера. Якщо прив'язати `AudioImporter.preset` до правила для текстур — `preset.ApplyTo()` поверне `false` і в Console з'явиться попередження.

## Стандартні правила

При першому запуску Unity автоматично створює `ImporterSettingsDatabase.asset` з такими правилами:

| Назва | Умова | Папка | Preset |
|-------|-------|-------|--------|
| UI Textures | `UI_` | `Assets/Art/UI/` | `TextureImporter_UI.preset` |
| General Textures | `T_` | `Assets/Art/Textures/` | `TextureImporter.preset` |
| Sound Effects | `SFX_` | `Assets/Audio/SFX/` | `AudioImporter.preset` |
| Music | `Mus_` | `Assets/Audio/Music/` | `AudioImporter_Music.preset` |

Всі правила можна змінити, видалити або доповнити через вікно.

## Структура файлів

```
Assets/AssetRouter/
├── Editor/
│   ├── Data/
│   │   ├── BaseImportRule.cs           ← абстрактний базовий клас: умови + targetFolder
│   │   ├── ImportRule.cs               ← конкретний клас: + preset
│   │   └── ImporterSettingsDatabase.cs ← ScriptableObject, контейнер всіх правил
│   ├── Logic/
│   │   ├── AssetMoveCandidate.cs       ← struct для пари (шлях, правило)
│   │   ├── DatabaseLocator.cs          ← знаходить і кешує SO
│   │   ├── RuleValidator.cs            ← перевіряє чи підходить файл під правило
│   │   ├── AssetRouterPostprocessor.cs ← перехоплює імпорт і виконує правила
│   │   └── AssetRouterInitializer.cs   ← створює SO при першому запуску
│   └── View/
│       └── AssetRouterWindow.cs        ← вікно Tools > Asset Router Settings
├── Presets/
│   ├── TextureImporter.preset
│   ├── TextureImporter_UI.preset
│   ├── AudioImporter.preset
│   └── AudioImporter_Music.preset
└── Tests/
    └── RuleValidatorTests.cs
```

## Архітектура для техліда

### Ієрархія даних

```
BaseImportRule      ← умови (prefix, suffix, extension, targetFolder) — абстрактний
    └── ImportRule  ← + preset (посилання на Unity Preset файл)
```

### SOLID

| Принцип | Як реалізовано |
|---------|----------------|
| **SRP** | Кожен клас має одну відповідальність: дані / UI / логіка / пошук / ініціалізація |
| **OCP** | Новий тип правила або preset — без змін в існуючих класах |
| **LSP** | `BaseImportRule` використовується скрізь, `ImportRule` підставляється без змін логіки |
| **DIP** | `Postprocessor` залежить від абстракції `BaseImportRule`, не від конкретного типу |

### Захист від нескінченного циклу імпорту

`AssetDatabase.MoveAsset` запускає повторний імпорт. Щоб уникнути повторного переміщення, система веде статичний `HashSet<string> AssetsBeingMoved`. Перед переміщенням цільовий шлях туди додається — при повторному імпорті він впізнається і пропускається.

### Кешування бази даних

`DatabaseLocator` кешує SO після першого `AssetDatabase.FindAssets`. Кеш скидається при збереженні через вікно або при створенні нового SO.

## Усунення неполадок

### Файл не переміщується

1. `Enable Auto Import` увімкнено у SO?
2. Розширення файлу є у списку `Monitored Extensions`?
3. Шлях файлу не починається з папки зі списку `Ignored Folders`?
4. Правило `Is Enabled`? Умови `Prefix` / `Extension Filter` правильні?
5. Перевір Console — кожна дія логується з префіксом `[AssetRouter]`

### Preset не застосовується

- Тип preset збігається з типом ассету? (TextureImporter preset → текстура, AudioImporter preset → аудіо)
- `Import Preset` поле у правилі не порожнє?
- Примусово переімпортуй файл: ПКМ → **Reimport**

### Попап не з'являється

- `Popup for Unknown Files` увімкнено у SO?
- Попап з'являється із затримкою — дочекайся завершення імпорту

### SO зник / "ImporterSettingsDatabase не знайдено"

- Зроби `Ctrl+R` у Unity — ініціалізатор створить SO знову
- Або: `Tools > Asset Router Settings` → **Create New**
